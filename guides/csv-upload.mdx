The Reprompt dashboard lets you upload a CSV file of places and enrich them in bulk without writing any code. Navigate to **[Place Enrichment → Upload CSV](https://app.repromptai.com/place-enrichment/upload-csv)** to get started.

<img src="/images/csv-upload-page.png" alt="CSV upload page in the Reprompt dashboard" />

## Prepare Your CSV

Your CSV must include a header row. The following columns are recognized:

| Column | Required | Description |
|--------|----------|-------------|
| `name` | Yes | Business or place name |
| `full_address` | Recommended | Full street address including city, state, zip |
| `latitude` | Optional | Latitude coordinate |
| `longitude` | Optional | Longitude coordinate |
| `place_id` | Optional | Your internal identifier for the place |

<Tip>
Including `full_address` alongside `name` significantly improves enrichment accuracy. If you have coordinates, include those too.
</Tip>

**Example CSV:**

```csv
place_id,name,full_address,latitude,longitude
store_001,Downtown Coffee,123 Main St New York NY 10001,40.7128,-74.0060
store_002,Airport Cafe,1 Airport Blvd Los Angeles CA 90045,33.9425,-118.4081
store_003,Lakeside Diner,456 Lake Shore Dr Chicago IL 60611,41.8902,-87.6164
```

## Row Limits

The upload page accepts CSV files up to **50,000 rows** per upload. If your file exceeds this limit, you'll need to split it before uploading.

## Splitting Large CSV Files

If your dataset has more than 50,000 rows, split it into smaller files first. Each chunk keeps the original header row.

### Using the command line

**macOS / Linux:**

```bash
# Split a large CSV into chunks of 50,000 rows each (preserving the header)
tail -n +2 large_dataset.csv | split -l 50000 - chunk_
for f in chunk_*; do
  head -1 large_dataset.csv > "split_${f}.csv"
  cat "$f" >> "split_${f}.csv"
  rm "$f"
done
```

**Windows (PowerShell):**

```powershell
$header = Get-Content large_dataset.csv -First 1
$rows = Get-Content large_dataset.csv | Select-Object -Skip 1
$chunkSize = 50000
$i = 0
for ($offset = 0; $offset -lt $rows.Count; $offset += $chunkSize) {
    $chunk = $rows[$offset..([Math]::Min($offset + $chunkSize - 1, $rows.Count - 1))]
    $i++
    @($header) + $chunk | Set-Content "split_part${i}.csv"
}
```

### Using Python

```python
import csv
import os

def split_csv(input_file, max_rows=50000):
    with open(input_file, 'r') as f:
        reader = csv.reader(f)
        header = next(reader)

        file_num = 1
        current_rows = []
        for row in reader:
            current_rows.append(row)
            if len(current_rows) >= max_rows:
                write_chunk(header, current_rows, input_file, file_num)
                file_num += 1
                current_rows = []

        if current_rows:
            write_chunk(header, current_rows, input_file, file_num)

def write_chunk(header, rows, original_name, part_num):
    base = os.path.splitext(original_name)[0]
    output = f"{base}_part{part_num}.csv"
    with open(output, 'w', newline='') as f:
        writer = csv.writer(f)
        writer.writerow(header)
        writer.writerows(rows)
    print(f"Wrote {len(rows)} rows to {output}")

split_csv("large_dataset.csv")
```

### Upload each chunk

Once split, upload each file separately through the dashboard. Each upload creates its own batch that you can monitor independently.

<Tip>
Name your batches descriptively (e.g. "Q1 Stores - Part 1 of 3") so you can track which chunks have completed.
</Tip>

## Monitoring Your Upload

After uploading, you'll be redirected to the batch detail page where you can track progress in real time.

<img src="/images/csv-upload-batch-status.png" alt="Batch status page showing enrichment progress" />

- **Pending** — jobs are queued and waiting to start
- **In Progress** — jobs are being enriched
- **Completed** — enrichment finished; results are available to download
- **Failed** — some jobs failed; you can reprocess them

See [Batch Processing](/guides/batch-processing) for details on monitoring, pagination, and reprocessing failed items.

## Downloading Results

## Best Practices

- **Include full addresses** — `full_address` is the single most impactful field for enrichment accuracy.
- **Add coordinates when available** — `latitude` and `longitude` help disambiguate places with similar names.
- **Use unique place IDs** — The `place_id` column lets you join enriched results back to your source data.
- **Split proactively** — If your dataset is close to 50K rows, split it. Smaller batches are easier to retry if something fails.
- **Start with a sample** — Upload 10–100 rows first to verify column mapping and attribute selection before processing your full dataset.
