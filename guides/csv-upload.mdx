The Reprompt dashboard lets you upload a CSV file of places and enrich them in bulk without writing any code. You can upload files with **up to 1 million rows** — the platform automatically splits them into batches for you.

Navigate to **[Datasets → Upload dataset](https://app.repromptai.com/place-enrichment/upload-csv)** to get started.

<Frame>
  <img src="/images/csv-upload-page.png" alt="CSV upload page in the Reprompt dashboard" />
</Frame>

Drag and drop your CSV file onto the upload area, or click to browse. The file must be under **200 MB**, include a **header row**, and contain at least a place name with an address or coordinates.

## Map Your Columns

After uploading, the dashboard parses your file and shows the **Map CSV Columns** screen. It auto-detects columns where possible.

<Frame>
  <img src="/images/csv-upload-column-mapping.png" alt="Essential field mapping after uploading a 1.29 million row CSV" />
</Frame>

### Essential Fields

These fields drive enrichment accuracy. Map as many as you have:

| Field | Required | Description |
|-------|----------|-------------|
| `id` | Yes | Unique identifier for each place |
| `name` | Yes | Business or place name |
| `latitude` | Recommended | Latitude coordinate |
| `longitude` | Recommended | Longitude coordinate |

### Additional Options

Expand **Additional Options** to map every column your CSV has. The more fields you map, the better your enrichment results.

<Frame>
  <img src="/images/csv-upload-additional-options.png" alt="Additional column mapping options including phone, website, address fields, and more" />
</Frame>

| Field | Description |
|-------|-------------|
| `wkt_geometry` | WKT geometry string |
| `phone` | Phone number |
| `website` | Official website URL |
| `category` | Place category |
| `opening_hours` | Opening hours (JSON) |
| `house` | House/building number |
| `street` | Street name |
| `city` | City |
| `postalCode` | Postal / zip code |
| `state` | State or province |
| `country_code` | ISO country code |
| `country` | Country name |

<Warning>
Map **every field you have** in your CSV. Unmapped columns are ignored during enrichment, which reduces match quality. For example, if your CSV has a `phone_number` column, make sure it's mapped to the `phone` field.
</Warning>

## Preview and Import

After mapping, the dashboard shows a **Preview** of your data. Click **Select All** to include every row, then review the import summary at the bottom.

<Frame>
  <img src="/images/csv-upload-preview.png" alt="Preview showing 1,290,391 records selected, ready to import as 259 datasets" />
</Frame>

The dashboard tells you exactly how many rows you're importing and how many datasets (batches) it will create. In this example, **1,290,391 rows** are automatically split into **259 datasets**.

Click **Import 1,290,391 rows as 259 datasets** to start the upload. The platform handles all the splitting for you — no need to manually chunk your file.

<Tip>
You can upload CSV files with up to **1,000,000 rows** in a single upload. The dashboard automatically splits them into batches of ~5,000 rows each.
</Tip>

## File Size Limits

| Limit | Value |
|-------|-------|
| Max file size | 200 MB |
| Max rows per upload | 1,000,000 |
| Auto-split batch size | ~5,000 rows |

If your CSV exceeds 200 MB, split it into smaller files before uploading. Each file can still contain up to 1 million rows as long as it's under the file size limit.

### Splitting files over 200 MB

For very large files that exceed the 200 MB file size limit, split them before uploading:

**macOS / Linux:**

```bash
# Split a large CSV into chunks of 500,000 rows each (preserving the header)
tail -n +2 large_dataset.csv | split -l 500000 - chunk_
for f in chunk_*; do
  head -1 large_dataset.csv > "split_${f}.csv"
  cat "$f" >> "split_${f}.csv"
  rm "$f"
done
```

**Python:**

```python
import csv, os

def split_csv(input_file, max_rows=500000):
    with open(input_file, "r") as f:
        reader = csv.reader(f)
        header = next(reader)
        file_num, current_rows = 1, []

        for row in reader:
            current_rows.append(row)
            if len(current_rows) >= max_rows:
                write_chunk(header, current_rows, input_file, file_num)
                file_num += 1
                current_rows = []
        if current_rows:
            write_chunk(header, current_rows, input_file, file_num)

def write_chunk(header, rows, original_name, part_num):
    base = os.path.splitext(original_name)[0]
    with open(f"{base}_part{part_num}.csv", "w", newline="") as f:
        writer = csv.writer(f)
        writer.writerow(header)
        writer.writerows(rows)

split_csv("large_dataset.csv")
```

Upload each resulting file separately through the dashboard.

## Monitoring Progress

After importing, each auto-split batch appears as a separate dataset in the sidebar under **Datasets**. You can track enrichment progress for each one independently.

- **Pending** — jobs are queued and waiting to start
- **In Progress** — jobs are actively being enriched
- **Completed** — enrichment finished; results are available
- **Failed** — some jobs failed; you can reprocess them

See [Batch Processing](/guides/batch-processing) for details on monitoring, pagination, and reprocessing failed items via the API.

## Best Practices

- **Map every column** — The more fields you map, the higher the enrichment accuracy. Don't leave columns unmapped if they contain useful data.
- **Include coordinates** — `latitude` and `longitude` help disambiguate places with similar names at different locations.
- **Give datasets descriptive names** — The dataset name helps you find results later. Name them by geography, date, or purpose (e.g. "texas" or "Q1 2025 Store Locations").
- **Start with a sample** — Upload a small subset (100–1,000 rows) first to verify column mapping before processing your full dataset.

## Large Uploads and Processing Time

Uploading and enriching large datasets (hundreds of thousands or millions of rows) can take a significant amount of time — both for the upload itself and for enrichment to complete across all batches.

If you're working with very large files or need a faster transfer method, you can use an **S3 bucket** to exchange data with Reprompt instead of uploading through the browser. Contact the Reprompt team to set up S3-based ingestion for your account.
