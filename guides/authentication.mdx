## Overview

The Reprompt API uses **Bearer token authentication** to secure all API endpoints. Each organization receives a unique API key that must be included in the `Authorization` header of every request. !

## Getting Your API Key

1. Log into your dashboard at [app.repromptai.com](https://app.repromptai.com)
2. Navigate to **Settings** \> **API Keys**
3. Copy your organization-specific API key
4. Keep your API key secure - treat it like a password

<Warning>
  Never expose your API key in client-side code, public repositories, or share it publicly. Always use environment variables to store your API key securely.
</Warning>

## Making Authenticated Requests

Include your API key in the `Authorization` header using the Bearer scheme:

<CodeGroup>

```bash cURL
curl -X POST \
  'https://reprompt--reprompt-fastapi-fastapi-app.us-west.modal.run/{org_slug}/place_enrichment/enrich' \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{
    "place_id": "place_123",
    "inputs": {
      "name": "Example Place",
      "latitude": 40.7128,
      "longitude": -74.0060
    }
  }'
```


```javascript Node.js
const response = await fetch(
  'https://reprompt--reprompt-fastapi-fastapi-app.us-west.modal.run/{org_slug}/place_enrichment/enrich',
  {
    method: 'POST',
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY',
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      place_id: 'place_123',
      inputs: {
        name: 'Example Place',
        latitude: 40.7128,
        longitude: -74.0060
      }
    })
  }
);

const data = await response.json();
console.log(data);
```


```python Python
import requests

response = requests.post(
    'https://reprompt--reprompt-fastapi-fastapi-app.us-west.modal.run/{org_slug}/place_enrichment/enrich',
    headers={
        'Authorization': 'Bearer YOUR_API_KEY',
        'Content-Type': 'application/json'
    },
    json={
        'place_id': 'place_123',
        'inputs': {
            'name': 'Example Place',
            'latitude': 40.7128,
            'longitude': -74.0060
        }
    }
)

data = response.json()
print(data)
```

</CodeGroup>

## Organization Slug

Most API endpoints require an `{org_slug}` parameter in the URL path. This is your organization's unique identifier, which you can find in your dashboard settings.

<Note>
  Replace `{org_slug}` in all API URLs with your actual organization slug, and replace `YOUR_API_KEY` with your actual API key.
</Note>

## Response Codes

- **200 OK** - Request successful
- **401 Unauthorized** - Missing or invalid API key
- **403 Forbidden** - API key doesn't have permission for this resource
- **429 Too Many Requests** - Rate limit exceeded

## Best Practices

<Steps>
  <Step title="Use Environment Variables">
    Store your API key in environment variables, not in your code:

    ```bash
    export REPROMPT_API_KEY="your_api_key_here"
    ```
  </Step>
  <Step title="Rotate Keys Regularly">
    Generate new API keys periodically and revoke old ones from your dashboard.
  </Step>
  <Step title="Use HTTPS Only">
    Always make API requests over HTTPS to ensure your API key is encrypted in transit.
  </Step>
  <Step title="Monitor Usage">
    Track your API usage in the dashboard to detect any unauthorized access.
  </Step>
</Steps>
