# AttributeSet

The `AttributeSet` enum defines predefined groups of attributes that can be enriched for a place. This allows you to specify the scope of enrichment without listing individual attributes.

## Usage Examples

### Quick Status Check

```json
{
  "place_id": "restaurant_001",
  "inputs": {
    "name": "Joe's Pizza",
    "latitude": 40.7359,
    "longitude": -73.9911
  },
  "attribute_set": "open_closed"
}
```

### Standard Business Information

```json
{
  "place_id": "business_456", 
  "inputs": {
    "name": "Central Cafe",
    "full_address": "123 Main St, New York, NY"
  },
  "attribute_set": "core"
}
```

### Complete Business Intelligence

```json
{
  "place_id": "detailed_analysis_789",
  "inputs": {
    "name": "The Italian Corner",
    "latitude": 40.7589,
    "longitude": -73.9851
  },
  "attribute_set": "all"
}
```

## Performance Comparison

| Attribute Set | Processing Time | Attributes | Cost | Best For |
|---------------|-----------------|------------|------|----------|
| `open_closed` | ~30 seconds | 1-2 | $ | Status validation |
| `core` | ~2-3 minutes | 5-8 | $$ | General info |
| `all` | ~5-8 minutes | 15+ | $$$ | Complete analysis |

## Alternative: Custom Attributes

Instead of using predefined sets, you can specify individual attributes:

```json
{
  "place_id": "custom_001",
  "inputs": { "name": "Business Name" },
  "attributes": ["website", "phone", "categories"]
}
```

<Note>
When both `attribute_set` and `attributes` are provided, the `attributes` array takes precedence.
</Note>
