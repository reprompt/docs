# JobListResponse

The `JobListResponse` model represents the response structure when retrieving a list of jobs, including pagination information and job summaries.

## Schema Definition

<Note>
This schema documentation is manually maintained but represents the structure defined in your OpenAPI specification. Keep this in sync with your Pydantic models.
</Note>

<ResponseField name="jobs" type="array<PlaceJobResult>" required>
  Array of job results. See [PlaceJobResult](/schemas/place-job-result) for individual job schema.
</ResponseField>

<ResponseField name="total_count" type="integer" required>
  Total number of jobs matching the query criteria.
  
  **Example:** `150`
</ResponseField>

<ResponseField name="page" type="integer" required>
  Current page number (1-based).
  
  **Example:** `1`
</ResponseField>

<ResponseField name="page_size" type="integer" required>
  Number of jobs per page.
  
  **Example:** `20`
</ResponseField>

<ResponseField name="total_pages" type="integer" required>
  Total number of pages available.
  
  **Example:** `8`
</ResponseField>

<ResponseField name="has_next" type="boolean" required>
  Whether there are more pages available.
  
  **Example:** `true`
</ResponseField>

<ResponseField name="has_previous" type="boolean" required>
  Whether there are previous pages available.
  
  **Example:** `false`
</ResponseField>
